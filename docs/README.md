## üìù Documentation Standards

### File Naming Conventions
- **Plans**: `YYYY-MM-DD-descriptive-name-plan.md`
- **Reports**: `YYYY-MM-DD-descriptive-name-report.md` 
- **Implementation Reports**: `YYYY-MM-DD-descriptive-implementation-report.md`
- **Architecture**: `descriptive-system-name-architecture.md`
- **Guides**: `YYYY-MM-DD-descriptive-name-guide.md` (when dated) or `descriptive-name-guide.md`

### Organization Principles
- **By User Type**: Guides organized for different user needs (users, developers, operators)
- **By Purpose**: Documents grouped by their primary function (analysis, implementation, planning)
- **By Date**: Plans and reports organized chronologically for easy tracking
- **Clear Names**: All files have descriptive, meaningful names reflecting their content
- **Logical Hierarchy**: Related documents grouped in appropriate subdirectories
